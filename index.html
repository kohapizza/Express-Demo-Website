"Taro was adversely affected by Hanako's crying" entails "Hanako cried." The indirect passive construction is the key inference step.

How to interact

  1. Wait for parsing to finish ‚Äî each sentence is assigned multiple CCG/DTS parse candidates, reflecting syntactic ambiguity.
  2. Explore parse candidates for the premise and hypothesis. lightblue allows ambiguity, so there are multiple proof paths ‚Äî and some combinations yield no proof.
  3. Run Typecheck on a parse. Parses with no unspecified arguments produce a single type-checked result; others may yield multiple.
  4. Run Prove to search for a proof that the hypothesis follows from the premise under the selected combination.
  5. The verdict (yes / no / unknown) appears at the top.
Hint for this problem: A proof exists when you choose Premise parse 1, Typecheck 1 + Hypothesis parse 1, Typecheck 1. Alternatively, Premise parse 4, Typecheck 1 + Hypothesis parse 1, Typecheck 1 also leads to a proof.

Requires Docker Desktop. Image: lilkohapizza/express.

Build & run ‚Äî from source

git clone https://github.com/DaisukeBekki/lightblue
cd lightblue

# Clone JSeM dataset
git clone https://github.com/DaisukeBekki/JSeM

# Install KWJA (morphological analyzer)
pip install kwja

# Build (first run takes ~30 min)
stack build

# Set environment variable (trailing slash required)
export LIGHTBLUE=$(pwd)/

# Launch Express
./lightblue jp jsem --jsemid 720 --nsample 1 \
  -s express \
  -f JSeM/data/v1.0/Verbs.xml \
  --nparse 4 --ntypecheck 4 --nproof 1 \
  --maxdepth 5 --depth 0

# ‚Üí http://localhost:3000/inference

How to use Express

A step-by-step walkthrough for first-time users. Express wraps lightblue's inference pipeline in an interactive shell ‚Äî you navigate it one branch at a time.

  1. 1

    Launch Express from the command line

    To run NLI inference, you need a JSeM-format XML file passed via the -f flag. This file defines the premise‚Äìhypothesis pairs Express will work on. Use --jsemid to select a specific problem from the file by its ID. Express starts a local web server and automatically opens it in your browser.

  2. 2

    The browser opens automatically

    Once Express starts, it automatically opens http://localhost:3000 in your browser. The page shows all CCG parse candidates for each sentence in the NLI problem ‚Äî premise sentences on the left, hypothesis on the right. Each candidate card displays its score, CCG syntactic type, and DTS semantic representation.

  3. 3

    Select a syntactic structure

    CCG/DTS parsing may produce multiple syntactic structures (e.g. due to prepositional phrase attachment ambiguity). Express displays each candidate as a collapsible card with its CCG type and DTS semantic representation. Click Typecheck on the structure you want to pursue.

  4. 4

    Select a type-checking outcome

    Type checking under Dependent Type Theory may yield multiple outcomes when a representation contains underspecified anaphoric terms. Each possible anaphora resolution is shown as a separate branch. Select the one that matches your intended reading (highlighted in blue), then click Proof Search.

  5. 5

    View the result ‚Äî and export if needed

    The theorem prover wani runs on the path you selected and returns YES, NO, or UNKNOWN. If the result is unexpected, click Query (pos) to export the exact proof search query as a file directly loadable by wani. Share this file with the theorem prover developer for collaborative debugging ‚Äî no need to manually reconstruct the query from scratch.
